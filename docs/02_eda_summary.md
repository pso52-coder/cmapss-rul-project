# EDA 요약 — NASA C-MAPSS FD001 (Step 02)

> 대상 데이터: `data/processed/train_with_rul.csv` (20,631행 × 27열, 엔진 100대)
> 노트북: `notebooks/01_eda.ipynb`
> **범위:** 데이터 이해와 유효 센서 후보 선별까지. 정규화·RUL 클리핑·모델링은 다음 단계로 분리(이 단계에서 변형 없음).

## 1. 엔진 수명(cycle 길이) 분포

| 항목 | 값 |
|---|---|
| 엔진 수 | 100 |
| 최소 수명 | 128 cycle |
| 최대 수명 | 362 cycle |
| 평균 수명 | 약 206 cycle |
| 중앙값 | 199 cycle |

→ 엔진마다 수명 편차가 크다(128~362). 모든 엔진을 동일 기준으로 다루기 어렵다는 뜻이며, RUL 클리핑 상한(예: 125)을 정할 때 이 분포가 근거가 된다.

## 2. RUL 분포

- 범위: 0 ~ 361, 평균 약 107.8
- RUL은 정의상 선형(`max_cycle − cycle`)이므로 0부터 최대 수명까지 퍼져 있다.
- **관찰된 문제:** RUL이 큰 구간(초기 사이클)은 실제 열화가 거의 없는데도 수명이 길게 라벨링된다. 초기 구간을 그대로 학습하면 모델이 비현실적 신호를 배우게 된다.
- → **다음 단계 근거:** Piecewise Linear RUL 클리핑(일정 값 이상은 상한으로 고정)이 필요하다. (이 단계에서는 적용하지 않음)

## 3. 센서 기초 통계

- 21개 센서의 평균·표준편차·범위가 제각각이다(예: 센서별 수백~수천 단위 차이).
- → **다음 단계 근거:** 모델 학습 전 정규화(Min-Max 또는 Z-score)가 필요하다.

## 4. 무변화 센서 식별 (핵심 산출물)

표준편차 기준으로 정보가 없는 센서를 제거 후보로 분류했다.

| 분류 | 센서 | 판단 |
|---|---|---|
| 완전 상수 (std = 0) | sensor_1, sensor_10, sensor_18, sensor_19 | 항상 동일값 → 정보 없음, 제거 |
| 준상수 (0 < std < 0.01) | sensor_5, sensor_6, sensor_16 | 거의 변화 없음 → 제거 후보 |

→ **제거 후보 총 7개.** 나머지 14개 센서가 유효 센서 후보. (실제 제거는 전처리 단계에서 수행)

## 5. 대표 엔진 센서 시계열

수명이 가장 짧은 엔진과 가장 긴 엔진을 비교했을 때, 정보가 있는 센서(sensor_2, 4, 7, 11, 12, 15 등)는 고장에 가까워질수록 단조 증가 또는 감소하는 **열화 추세**를 보인다. 이는 RUL 예측에 쓸 수 있는 신호가 실제로 존재함을 시각적으로 확인한 것이다.

## 6. RUL-센서 상관관계

RUL과 선형 상관이 높은 센서(절댓값 기준 상위):

| 순위 | 센서 | 상관계수 |
|---|---|---|
| 1 | sensor_11 | -0.696 |
| 2 | sensor_4 | -0.679 |
| 3 | sensor_12 | +0.672 |
| 4 | sensor_7 | +0.657 |
| 5 | sensor_15 | -0.643 |
| 6 | sensor_21 | +0.636 |
| 7 | sensor_20 | +0.629 |
| 8 | sensor_2 | -0.606 |

→ 이 센서들이 모델의 핵심 피처 후보다.

> **상관계수 해석의 한계:** 상관계수는 유효 센서 후보를 판단하는 **참고 지표**로만 사용하고, 시계열 추세와 모델 기반 중요도 해석을 함께 확인해야 한다. C-MAPSS는 독립 행 데이터가 아니라 엔진별 시계열이므로 단순 상관계수가 (1) 엔진별 개체 차이, (2) cycle 진행에 따른 시간 구조, (3) 비선형 관계, (4) 운전조건과 열화의 분리를 충분히 반영하지 못한다.
