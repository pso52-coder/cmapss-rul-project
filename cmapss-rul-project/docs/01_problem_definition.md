# 문제 정의서 — 터보팬 엔진 잔여 수명(RUL) 예측

> NASA C-MAPSS FD001 데이터셋 기반. 02 교안의 단계적 접근(Step 1~5)과 좋은 문제 정의의 3요소(비즈니스 목표 / ML 목표 / 성공 기준)를 함께 채운다.

## 1. 비즈니스 목표

- **누가 이 문제를 풀어달라고 했는가?**: 항공사 정비 계획 담당자 및 엔지니어링 팀.
- **지금 무엇이 문제인가?**: 운항 중인 터보팬 엔진의 고장 시점을 사전에 알 수 없어, 과도한 예방 정비(비용 낭비) 또는 예상치 못한 결함(결항·안전사고)이 발생하고 있다. 매년 수십억 원 규모의 손실이 발생한다.
- **예측이 맞으면 무엇이 좋아지는가?**: 엔진별 잔여 수명을 미리 알면 정비 시점을 적시에 잡아 불필요한 예방 정비 비용을 줄이고, 고장으로 인한 결항·안전사고를 예방할 수 있다.

### 오판정 비용 — 무엇이 더 위험한가

| 오류 유형 | 의미 | 결과 | 위험도 |
|---|---|---|---|
| **False Negative** | 고장 임박을 정상으로 예측 (과대 예측) | 엔진 고장 미탐지 → 결항·인명사고 | **압도적으로 높음** |
| False Positive | 정상을 고장 임박으로 예측 (과소 예측) | 과잉 정비 | 비용 손실에 그침 |

→ **고장을 놓치는 False Negative가 훨씬 위험**하므로, 모델은 RUL을 **보수적으로(낮게) 예측**하는 방향을 허용하고, 평가 지표 해석에서 FN을 최우선으로 본다.

## 2. ML 목표

- **입력**: 엔진 ID, 사이클 번호, 운전 조건 3개(operational setting 1~3), 센서 측정값 21개의 시계열.
- **출력**: 각 시점에서 고장까지 남은 사이클 수 — **RUL (연속형 수치)**. 추가로 RUL ≤ 30 구간의 고장 임박 여부(이진 분류).
- **한 문장 정의**: "엔진의 센서·운전조건 시계열을 입력받아 현재 시점의 잔여 수명(RUL, 사이클 수)을 회귀로 예측하고, RUL이 30 사이클 이하이면 조기 경보를 발령한다."
- **문제 유형**: **회귀(Regression)** 를 메인으로, 조기 경보는 분류를 병행.
  - **선택 근거**: 담당자에게 필요한 것은 "고장까지 몇 사이클 남았는가"라는 구체적 수치이며, 이를 기반으로 정비 일정을 역산해야 한다. 단순 이진 분류(고장/정상)만으로는 정비 계획 수립에 필요한 정보가 부족하다.

## 3. 성공 기준

| 구분 | 지표 | 기준값 | 근거 |
|---|---|---|---|
| 기술 기준 | RMSE | ≤ 30 사이클 | 정비 계획 수립이 가능한 최소 신뢰 범위 |
| 기술 기준 | NASA Score Function | 최소화 | 늦은 예측(과대 예측)에 더 큰 패널티 부여 |
| 비즈니스 기준 | 조기 경보 정확도 | ≥ 85% | RUL ≤ 30 구간 내 고장 임박 감지율 |
| 비즈니스 기준 | False Negative Rate | ≤ 5% | 고장 임박을 정상으로 분류하는 오류 최소화 |

- **기준의 근거**: 엔진 1대 조기 정비 비용 대비 미탐지 고장 비용(리콜·안전사고)의 비대칭성을 운영상 허용 오차로 환산해 RMSE 임계치를 역산. NASA C-MAPSS는 늦은 예측에 비대칭 패널티를 주는 Score Function이 표준 평가 지표로 함께 쓰인다.

## 4. 데이터 설명

- **출처**: NASA C-MAPSS Turbofan Engine Degradation Dataset, 하위 세트 **FD001**.
- **규모**: Train 100개 엔진 / 20,631행, Test 100개 엔진 / 13,096행. 컬럼 26개(엔진 ID 1, 사이클 1, 운전조건 3, 센서 21). 정답 RUL은 `RUL_FD001.txt`로 별도 제공(100개).
- **타깃 변수**: RUL(잔여 수명, 사이클 수). Train은 고장까지 기록되므로 `엔진별 max_cycle − 현재 cycle`로 직접 생성한다.
- **조건/고장 모드**: 단일 운전 조건(Sea Level), 단일 고장 모드(HPC Degradation).
- **한계 및 대응**:

| 문제 유형 | 내용 | 대응 방안 |
|---|---|---|
| 센서 중복 | 21개 중 일부는 거의 변화 없음(상수에 가까움) | 분산 기반 피처 선택으로 제거 |
| RUL 레이블 정의 | 초기 사이클은 열화가 거의 없어 선형 RUL 가정이 비현실적 | Piecewise Linear RUL(최대값 클리핑) 적용 |
| 스케일 불균형 | 센서별 단위·범위 상이 | Min-Max 또는 Z-score 정규화 |
| 운전 조건 단일 | 실제 다양한 환경 대표성 부족 | 향후 FD002~004로 확장 검토 |

## 5. 단계 분해 (핵심)

```
센서 데이터로 엔진의 남은 수명을 예측
  → ① 원본 데이터 로딩 (sep=r'\s+', 헤더 없음, 26컬럼)
     → ② 데이터 구조 검증 (shape·컬럼·NaN 확인)
        → ③ RUL 타깃 생성 (max_cycle − cycle)
           → ④ 센서 기초 통계 확인 (분포·분산·추세)
              → ⑤ 유효 센서 선별 (무변화 센서 제거)
                 → ⑥ 전처리 및 정규화 (Piecewise Linear RUL, 스케일링)
                    → ⑦ RUL 예측 모델링 (회귀)
                       → ⑧ 조기 경보 판단 (RUL ≤ 30: 분류)
                          → ⑨ 정비 의사결정 지원 (우선순위·권장 시점)
```

| 단계 | Input | Output | 목적 |
|---|---|---|---|
| 1. 원본 데이터 로딩 | FD001 원시 txt 3개 | 컬럼명 부여된 DataFrame (26컬럼) | 함정(헤더 없음·행끝 공백) 없이 데이터를 정확히 적재 |
| 2. 데이터 구조 검증 | 로딩된 DataFrame | 검증 결과(shape·컬럼·NaN·엔진 수) | 이후 단계의 전제가 깨지지 않도록 조기에 오류 차단 |
| 3. RUL 타깃 생성 | 엔진별 max_cycle, 현재 cycle | 사이클별 RUL 수치 컬럼 | 예측 대상(타깃) 확정 및 정답 라벨 확보 |
| 4. 센서 기초 통계 확인 | 21개 센서 시계열 | 센서별 분포·분산·열화 추세 | 어떤 센서가 정보(열화 신호)를 담는지 파악 |
| 5. 유효 센서 선별 | 센서 기초 통계 | 유효 센서 N개 목록 | 상수에 가까운 무변화 센서 제거로 노이즈 축소 |
| 6. 전처리 및 정규화 | 유효 센서값, RUL | 정규화 시계열 + Piecewise Linear RUL | 스케일 통일 + 초기 구간의 비현실적 선형 RUL 보정 |
| 7. RUL 예측 모델링 | 전처리된 피처 벡터 | 예측 RUL(연속형) | 정비 일정 역산에 필요한 잔여 수명 수치 산출 |
| 8. 조기 경보 판단 | 예측 RUL | 고장 임박 여부(RUL ≤ 30) | FN(고장 누락)을 최소화한 위험 감지 |
| 9. 정비 의사결정 지원 | 경보 + 예측 RUL | 정비 우선순위·권장 정비 시점 | 현장 담당자의 정비 계획 수립을 직접 지원 |
